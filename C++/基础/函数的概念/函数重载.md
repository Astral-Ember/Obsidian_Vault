函数重载

1. 基本概念
2. 实例

重载概念  
在同一作用域中（比如全局作用域，以后学的结构体、类中）  
函数名（方法）名相同  
参数的数量不同  
或者  
参数的数量相同，但参数的类型或顺序不同
 
作用：  
命名一组功能相似的函数，减少函数名的数量，避免命名空间的污染  
提升程序可读性
 
说人话：  
同一作用域内  
函数名相同  
参数数量不同  
参数类型不同  
参数顺序不同  
就称为函数重载，或者称这样的函数为重载函数
 
要点：  
重载和返回值类型无关，只和参数类型、个数、顺序有关系  
调用时 程序会自己根据传入的参数类型判断使用哪一个重载

|   |
|---|
|int calcSum(int a, int b)  <br>{  <br>return a + b;  <br>}|

参数数量不同

|   |
|---|
|int calcSum(int d, int e, int c)  <br>{  <br>return d + e + c;  <br>}|

数量相同 类型不同

|   |
|---|
|float calcSum(float a, float b)  <br>{  <br>return a + b;  <br>}|

数量相同 顺序不同

|   |
|---|
|double calcSum(float a, double b)  <br>{  <br>return a + b;  <br>}<br><br>  <br><br>double calcSum( double b, float a)  <br>{  <br>return a + b;  <br>}|

注意：默认参数（可选参数）会影响重载，编译器会不清楚该选择哪个重载  
在使用重载函数时，一定要先声明，如果忘记声明，可能调用的函数不会是你需要的函数
