字符数组

1. 字符数组的声明和初始化
2. 使用字符串初始化字符数组
3. 利用结束符号决定字符数组表示的字符串何时结束

字符数组就是类型为字符串char的一维数组  
聚合初始化：

|   |
|---|
|char charArr[5] = { 'a', 'b', 'c', 'd', 'e' };  <br>char charArr1[] = { 'a', 'b', 'c', 'd', 'e' };|

单独初始化

|   |
|---|
|char charArr2[5];  <br>charArr2[0] = 'a';  <br>charArr2[1] = 'b';|

这些几种声明初始化的方式，如果直接将字符数组拿来打印，会出现问题

![Exported image](Exported%20image%2020251125135844-0.png)  

可以直接使用字符串对字符数组进行赋值

|   |
|---|
|char charArray[] = "Hello, World!";  <br>cout \<\< charArray \<\< endl;  <br>int length = sizeof(charArray) / sizeof(char);  <br>cout \<\< length \<\< endl;//14  <br>//原本看起来只有13个字符，但实际上多了一个结束符'\0'，所以长度是14  <br>//通过观察长度可以发现，字符串字面值在内存中是以字符数组的形式存储的，并且以空字符'\0'结尾，以标志字符串的结束。|
 
如果不对字符数组后面的内容进行初始化，并且其中没有 \0  
在打印时就不知道何时结束，就会不停的往后读取  
因为没有明确的终止符，程序会根据内存中是否存在 ‘\0’ 停止打印  
可能打印出多余字符，甚至是乱码  
也就是说可能出现访问到字符数组范围之外的内存区域，产生未定义的行为  
关于 “烫” 字符  
在内存中，汉字字符使用多字节编码（例如 UTF-8 或 GBK）  
当程序读取到这些内存内容时，会尝试解释它们  
由于编码规则，有可能形成不合法的汉字编码，导致控制台显示异常

|   |
|---|
|char charArr[6] = { 'a', 'b', 'c', 'd', 'e','\0'};  <br>cout \<\< charArr \<\< '\n';|

这样子就好啦:)
