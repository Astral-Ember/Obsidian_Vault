指针的基础使用

1. 指针的声明
2. 指针的初始化
3. 指针的解引用操作符
4. 空指针和野指针
5. 总结：

声明写法：  
变量类型* 指针变量名；  
或  
变量类型 * 指针变量名；  
注意：  
任何变量类型都可以声明为指针变量类型  
就是在变量类型后面加一个 *  
指针变量的常用命名规范  
前缀法：p 或 ptr 作为前缀  
后缀法：_p 或 ptr 作为后缀  
指针类型变量名：ptr p+变量类型+变量名 =\> ptrIntValue, pIntValue 等等

|   |
|---|
|int* p_i;  <br>char* p_c;  <br>string* p_s;|

指针必须赋值为内存中的有效地址（已经分配给变量的地址）  
也就是说，根据我们当前学习的知识，当使用指针变量时  
必须让它指向一个已经声明了的变量，并且该变量的类型需要和指针变量的变量类型一致  
p.s.  
为什么要用相同的变量类型的指针去存储地址  
主要是因为一个变量它所占用的内存单元可能是多个，而指针变量存储的是它第一个内存单元的门牌号  
如果指针变量类型和存储的变量类型不一致，那么之后就无法准确的读取对应多个内存单元来获取其中的内容  
如果我们想声明一个指针指向 i 的内存地址，我们需要获得 i 的地址  
我们可以利用 地址符号 & i 来取得  
继续强调：要访问指针指向的内存单元，必须知道该内存单元的地址

|   |
|---|
|int i = 10;  <br>//int* p = &i;  <br>int* p;  <br>p = &i;//这种写法也可以  <br>cout \<\< p;|

所谓的 指针解引用操作符，其实就是 *  
如果我们想获取到指针所指向的内存地址中存储的值  
我们只需要在指针变量前加一个解引用操作符，便可以得到其中存储的值

|   |
|---|
|int i = 10;  <br>int* p = &i;  <br>cout \<\< *p;//10|

并且我们还可以利用它改变原有的值

|   |
|---|
|int i = 10;  <br>int* p = &i;  <br>*p = 20;//解引用  <br>cout \<\< *p \<\< '\n';//20  <br>cout \<\< i;//20|

p.s.为什么i原本的值改变了？  
解引用改变的是指针所指向的内存中的值，而不是指针变量本身的地址，因此会改变原本变量的值。  
空指针是  
不指向任何有效内存的指针  
给指针赋值为 nullptr 或 NULL（老版本）表示该指针为空指针

|   |
|---|
|int* p = nullptr;|

野指针是  
指向无效或不可预测内存地址的指针  
野指针会导致程序出现各种不可预料的错误  
比如：程序崩溃、数据损坏、访问非法内存等等  
主要出现的原因：  
未初始化的指针（目前主要遇到的为该种情况）  
指向内存被释放后仍使用的指针  
指向内存位置超出作用域的指针  
指向未定义为分配的内存的指针  
等等  
注意：  
为了避免出问题  
在声明指针时如果不马上使用，建议将指针声明为空指针  
指针就是存储变量内存地址的变量  
想要获取某一个变量地址使用 地址符号 & 来获取  
想要获取指针指向内存地址中存储的值 使用解引用操作符 * 取值  
在声明指针时，如果不马上传使用该指针，建议大家将其声明为空指针 null ptr / NULL（老版本）表示该指针为空指针  
野指针会对程序带来不可预估的问题，需要引起重视
 
对于 * 和 & 的记忆口诀  
看见 & 想地址  
看见 * 想内容
