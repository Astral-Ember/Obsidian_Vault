1. 结构体和类的区别
2. 开发时如何去选择
3. 既然类似 为何保留？

C++ 中的结构体（struct）和类（class）几乎是一模一样的  
唯一的区别就是默认为访问权限不同

1. 类的默认访问权限是 private 私有的

结构体的默认访问权限是 public 公共的

3. 类继承时，默认继承权限是 private 的（若未指定则按默认）

结构体继承时，默认继承权限是 public 的

除此之外结构体和类功能上完全相同  
我们目前学习的所有面向对象相关知识都可以用在结构体上  
虽然在 C++ 中结构体和类几乎一样  
但是我们在实际开发时还是会有约定俗成的规则

1. 只包装含数据的简单数据结构，推荐使用 结构体
2. 更符合面向对象思想的类型，推荐使用 类

C++ 语言是基于 C 语言设计的  
结构体是 C 语言中的内容  
C++ 作者本贾尼·斯特劳斯特卢普（Bjarne Stroustrup）在设计 C++ 时  
他的目标是在保留 C 的基础上，加上面向对象的能力  
因此他们选择不废弃结构体，而是拓展它，让它可以做“类”的事情  
这样做的好处是，可以向下兼容，老的 C 代码可以在 C++ 中使用
