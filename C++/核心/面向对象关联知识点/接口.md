接口其实是 C# 和 Java 中的一个关键词 interface  
它是一种行为的抽象规范  
我们可以继承接口来表示行为的继承  
但是在 C++ 中并没有专门的接口关键词  
因此，为了表示行为的继承，我们经常需要在 C++ 中模拟接口  
什么是行为的抽象  
比如：飞机、乌、超级赛亚人、超人  
他们其实不属于同一种类型，但是都会飞  
如果我们用面向对象的思想，其实不太好将他们归纳出一个基类  
因此我们往往会把他们的行为抽象出来，声明一个表示行为的接口基类
 
C++ 中实现接口的本质：  
只有[纯虚函数的抽象类](抽象类.md)  
注意：

1. 接口类命名规范为： I 类名
2. 接口类中建议只包含纯虚函数和虚析构函数，不建议包含成员变量和普通函数
3. 接口类一般只需要 .h，因为不需要实现

代表该类是一个接口（Interface）  
注意，这是一种比较通用的命名规范  
可以使用里氏替换 把不同种类但是有相同行为的对象存储在一起进行统一的管理  
接口类继承接口类时，不需要去实现  
待对象类继承接口后 类自己去实现所有内容即可
