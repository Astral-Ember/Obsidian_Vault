成员变量和访问修饰符

1. 知识回顾：[类和对象](类和对象.md)
2. 成员变量
3. 访问修饰符
4. 成员变量的使用和初始值

基本规则  
声明在类语句块中  
用来描述对象的特征  
可以是任意变量类型  
数量不做限制  
是否赋值根据需求来定（老版本 C++ 不能直接在声明时赋值，只能在构造函数中赋值）  
不能有非指针类型的自己类型的成员 否则会出现构造函数递归 造成死循环  
但是如果存在类以外的类，可以用指针调用

|   |
|---|
|enum E_sex  <br>{  <br>man,  <br>woman  <br>};  <br>struct Position  <br>{  <br>int x;  <br>int y;  <br>Position(int x, int y)  <br>{  <br>this-\>x = x;  <br>this-\>y = y;  <br>}  <br>};  <br>class Person  <br>{  <br>//成员变量 用来描述 人 这种对象的特征  <br>//写在类语句块中  <br>string name = "forward";  <br>int age;  <br>E_sex sex = man;  <br>Position pos = Position(0, 0);  <br>//Person mother;不能有非指针类型的自己类型的成员 否则会出现构造函数递归 造成死循环  <br>Person* father = nullptr;  <br>Person* friends[10];  <br>};|

public — 公共的 自己（内部）和别人（外部）都能访问和使用（完全公开，任何地方都能访问）  
private — 私有的 自己（内部）才能访问和使用 不写访问修饰符时 默认为private（严格有，只有当前类内部才能访问）  
protected — 保护的 自己（内部）和子类才能访问和使用（对自己和子类开放，外部不可访问）  
决定类内部的成员 的 访问权限
 
使用：

初始化：  
一定要对类中的成员变量进行初始化

1. 在声明时直接初始化
2. 在构造函数中初始化

（如果都初始化了，会以构造函数中初始化的为准）  
注意：  
如果我们不对成员变量进行初始化 那么它们的初始值会是一种不确定的状态  
我们一定要避免这种情况 因此我们需要对它们进行初始化
