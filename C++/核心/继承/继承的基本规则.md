回顾访问修饰符作用  
public 一 公共的 自己(内部)和别人(外部)都能够访问和使用（完全公开，任何地方都能访问）  
private 一 私有的 自己(内部)才能访问和使用 不写 默认为 private（严格私有，只有当前类成员才能访问）  
protected 一 保护的 自己(内部)和子类才能访问和使用（对自己和子类开放，外部不可访问）  
决定类内部的成员 的 访问权限

C++ 中允许子类存在和父类同名的成员  
但是不建议这样使用  
子类同名成员会覆盖父类成员  
也可以通过作用域解析符 :: 来强制访问父类成员  
例如：父类名::成员  
子类同名不同参数成员函数会隐藏父类成员  
当子类定义了一个和父类同名但参数不同的函数时  
父类中所有同名函数都会被隐藏，即使参数不同也不能再直接访问  
可以在子类中通过 using 父类名::父类方法名 来解决隐藏问题  
如果我们不希望一个类可以被继承  
我们可以通过final关键词来修饰类  
基本语法  
class 类名 final  
{  
}  
在 类名后加上final关键字 就可以禁止改类被继承了
