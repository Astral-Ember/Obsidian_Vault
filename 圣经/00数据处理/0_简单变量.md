## 1. 整型
1. short (int)
	   用于表示短整数，通常占用16位
	   存储范围：-32768~32767	   	   
2. int
	   用于表示整数，通常占用32位，至少与short一样长
	   存储范围：-21亿多~21亿多
3. long
	   用于表示长整数，通常占用32位，且至少与int一样长
	   存储范围：-21亿多~21亿多 或 -9223亿多~ 9223亿亿多
	   不同的原因是因为操作系统不同
4. long long
	   用于表示更长的整数，通常占用64位，且至少与long一样长
	   存储范围：-9223亿多~ 9223亿亿多
## 2. 无符号类型

1. unsigned short
   存储范围：0~65535
2. unsigned int 
   存储范围：0~42亿多
3. unsigned long 
   存储范围：0~42亿多或0~18466亿亿多
4. unsigned long long
    存储范围：0~42亿多或0~18466亿亿多
==注意：==
变量的溢出就像里程表一样，如果超越了限制，其值将为范围另一端的取值
C++确保了无符号类型的这种行为
C++并不保证有符号型变量超越限制（上溢和下溢）时不出错
以整形的溢出为例：
有符号：  
+1: 0 -> 32767 此时再加1则溢出了, 数据变为: -32768 -> 0  
-1: 0 -> -32768 此时再减1则溢出了, 数据变为: 32767 -> 0  
即：正数最大值继续增大，则会溢出到负数最小值；负数最小值继续减小，则会溢出到正数最大值
无符号： 
+1: 0 -> 65535 此时再加1则溢出了, 数据变为: 0 -> 65535
-1: 此时减1则溢出了, 数据变为: 65535 -> 0
即：数据在0到65535之间循环
## 3. char类型：字符和小整数

`char ch = 'a'`
char：用于表示字符，通常占用1个字节。
 wchar_t：用于表示宽字符，通常占用2或4个字节
char16_t：用于表示16位Unicode字符，占用2个字节
char32_t：用于表示32位Unicode字符，占用4个字节
创建字符型变量时单引号内只能有一个字符
字符型变量并不是把字符本身放到内存中存储，而是将对应的ASCII编码放入存储单元

## 4. bool类型

bool true false 表示真假的数据类型
true可以代表1 false表示0
非零可以表示true，零可以表示false
