数据存储的3个重要属性

- 存储的值为多少
- 存储的信息是什么类型
处理存储数据的新策略刚好相反，将地址视为指定的量，而将值视为派生量。一种特殊类型的变量——指针用于存储数据的地址。因此指针名表示的是地址。* 运算符被称为“解（除）引用”运算符，将其用于指针就可以得到该段地址存储的值


## 1.声明和初始化指针
指针声明必须指定指针指向的数据的类型。
`数据类型* 指针名`
对每个指针变量名，都需要使用一个*
`int* p1,p2;`
这表明创建一个int指针p1和int变量p2

可以在声明语句中初始化指针。在这种情况下，被初始化的是指针，而不是它指向的值。也就是说，下面的语句将pt(而不是* pt)的值设置为&higgens:
```
int higgens = 5;
int* pt = &higgens;
```
也就是说声明的指针变量记录的永远是地址

## 2.指针的危险性
在C++中创建指针时，计算机将分配用来存储地址的内存，但不会分配用来存储指针所指向的数据的内存。为数据提供空间是一个独立的步骤，如下所示:
`long * fellow;
`*fellow = 223323;`
fellow是一个指针，但是不知道指向哪里
==总的来说：一定要在对指针应用解除引用运算符(* )之前，将指针初始化为一个确定的、适当的地址。==


## 3.使用new来分配内存
在C++中可以用new来动态分配内存，由程序员申请并释放
为一个数据对象(可以是结构，也可以是基本类型)获得并指定分配内存的通用格式如下:
`typeName * pointer name = new typeName;`

==注意==：new分配的内存块通常与常规变量声明分配的内存块不同，常规变量声明分配的内存是在栈（stack）内存中；而new分配的内存是在堆（heap）内存中


## 4.使用delete释放内存
当需要内存时，可以使用new来请求，但是使用完内存后一定要使用delete运算符释放内存
```
int * ps = new int; // allocate memory with new
...// use the memory
delete ps;// free memory with delete when done
```
这将释放ps指向的内存，但不会删除指针ps本身

==注意==：不要尝试释放已经释放的内存块；只能用delete 来释放使用new分配的内存，然而，对空指针使用delete是安全的


## 5.使用new创建动态数组
如果程序只需要一个值，则可能会声明一个简单变量，因为对于管理一个小型数据对象来说，这样做比使用new和指针更简单。通常,对于大型数据(如数组、字符串和结构)，应使用new，这正是new的用武之地。例如，假设要编写一个程序，它是否需要数组取决于运行时用户提供的信息。如果通过声明来创建数组，则在程序被编译时将为它分配内存空间。不管程序最终是否使用数组，数组都在那里，它一直占用了内存
#### 1、使用new创建动态数组
在C++中，创建动态数组只要将数组的元素类型和元素数目告诉new即可。必须在类型名后加上方括号，其中包含元素数目。例如,要创建一个包含10个int元素的数组，可以这样做:
`	int* ptr = new int[10];`
