MI（multiple inheritance，多重继承）描述的是有多个直接基类的类。与单继承一样，公有MI表示的也是is-a关系。例如，可以从Waiter类和 Singer 类派生出 SingingWaiter类：
`class SingingWaiter : public Waiter, public Singer { ... };`
注意：必须使用关键字public来限定每一个基类。这是因为，除非特别指出，否则编译器将认为是私有派生：
`class SingingWaiter : public Waiter, Singer { ... }; // Singer is a private base`

MI可能会给程序员带来很多新问题。其中两个主要的问题是：
- 从两个不同的基类继承同名方法；
- 从两个或更多相关基类那里继承同一个类的多个实例。
为解决这些问题，需要使用一些新规则和不同的语法。

==注意==：多重继承可能导致函数调用的二义性：派生类可能从两个父类中继承两个完全不同的同名方法，因此可以使用类名和作用域解析运算符显式的指出使用哪个函数
```
SingingWaiter newhire ("Elise Hawks", 2005, 6, soprano);
newhire.Singer::Show(); // use Singer version
```
当然更好的方法是在派生类中重新定义函数，并指出要使用的函数
```
void SingingWaiter::Show()
{
	Singer::Show() ;
}
```

## 菱形继承
菱形继承是多继承的一种：
```
class A
{
public:
	A(int data) :ma(data) { cout << "A()" << endl; }
	~A() { cout << "~A()" << endl; }
protected:
	int ma;
};
class B :public A
{
public:
	B(int data) :A(data), mb(data) { cout << "B()" << endl; }
	~B() { cout << "~B()" << endl; }
protected:
	int mb;
};
class C :public A
{
public:
	C(int data) :A(data), mc(data) { cout << "C()" << endl; }
	~C() { cout << "~C()" << endl; }
protected:
	int mc;
};
class D :public B, public C
{
public:
	D(int data) : B(data), C(data), md(data) { cout << "D()" << endl; }
	~D() { cout << "~D()" << endl; }
protected:
	int md;
};
int main()
{
	D d(10);

	return 0;
}

```
运行的结果是：对于基类A而言，构造了两次，析构了两次
因此，多重继承的主要问题是：通过多重继承，有可能得到重复的基类数据，并且可能重复的构造和析构同一个基类对象
为了避免重复现象产生可以使用：**虚基类**
以下代码示例
```
class A
{
public:
	A(int data) :ma(data) { cout << "A()" << endl; }
	~A() { cout << "~A()" << endl; }
protected:
	int ma;
};
class B :virtual public A
{
public:
	B(int data) :A(data), mb(data) { cout << "B()" << endl; }
	~B() { cout << "~B()" << endl; }
protected:
	int mb;
};
class C :virtual public A
{
public:
	C(int data) :A(data), mc(data) { cout << "C()" << endl; }
	~C() { cout << "~C()" << endl; }
protected:
	int mc;
};
class D :public B, public C
{
public:
	D(int data) :A(data), B(data), C(data), md(data) { cout << "D()" << endl; }
	~D() { cout << "~D()" << endl; }
protected:
	int md;
};

```
注意区分：[[6_抽象基类|抽象基类]]是指有纯虚函数的类，而虚基类是指被虚继承的类
改成虚继承，A就成为了虚基类
采用虚继承，把有关重复的基类A改为虚基类，这样的话，对于A构造的任务就落到了最终派生类D的头上，因此，在D的构造函数的初始化成员列表中要调用A的构造函数

总之，虚继承可以解决多重继承的问题，虚继承底层实现原理与编译器相关，一般通过虚基类指针和虚基类表实现，**每个虚继承的子类都有一个虚基类指针（占用一个指针的存储空间，4字节）和虚基类表（不占用类对象的存储空间）**（需要强调的是，虚基类依旧会在子类里面存在拷贝，只是仅仅最多存在一份而已，并不是不在子类里面了）；当虚继承的子类被当做父类继承时，虚基类指针也会被继承。
间接性表明了在访问虚基类的成员时同样也必须通过某种间接机制来完成，共享性表象在虚基类会在虚继承体系中被共享，而不会出现多份拷贝。
